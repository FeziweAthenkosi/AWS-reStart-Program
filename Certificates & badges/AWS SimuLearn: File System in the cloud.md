# AWS SimuLearn: File System in the Cloud

## 📖 Overview
This project documents my completion of the **AWS SimuLearn: File System in the Cloud**. In this simulation, I gained hands-on experience in architecting, configuring, and managing scalable, high-performance file storage solutions within the AWS Cloud environment.

## 💡 What I Learned
During this SimuLearn, I focused on understanding the differences between object, block, and file storage, and specifically how to implement managed file systems to support distributed applications. Key takeaways include:

*   **Network File System (NFS) Implementation:** Configuring mount targets to allow multiple EC2 instances to access a shared file system.
*   **Scalability & Performance:** Understanding how managed file systems automatically scale capacity and throughput based on demand.
*   **Security & Access Control:** Implementing Security Groups to restrict access to file systems and ensuring data is protected at rest and in transit.
*   **Lifecycle Management:** Managing file system data and optimizing costs through automated lifecycle policies.

## 🛠️ AWS Services Used
*   **Amazon EFS (Elastic File System):** Used as the primary scalable file storage solution.
*   **Amazon EC2:** Provisioned compute instances to act as clients for the file system.
*   **Amazon VPC:** Configured networking components (Subnets, Route Tables) to ensure secure connectivity.
*   **AWS Security Groups:** Implemented stateful firewall rules to control traffic between EC2 and EFS.
*   **IAM (Identity and Access Management):** Managed permissions for secure service interaction.

## 🚀 Skills Enhanced
*   **Cloud Architecture:** Designing storage solutions that are highly available and durable.
*   **Network Configuration:** Mastering the interaction between private subnets and managed storage services.
*   **Troubleshooting:** Identifying and resolving connectivity issues between compute resources and file systems.
